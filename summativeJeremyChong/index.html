The Mailman

By Jeremy Chong

Premise of the Project

For this summative project, I plan to make a 2D top-down platformer, similar to the arcade game Frogger. The main objective of this game is to send mail to all of the houses by navigating the player to each home while avoiding obstacles such as cars. Once the player has successfully done this, the difficulty of the game will then increase. Once the player loses a certain amount of lives or runs out of time, the game will be over.

Work Distribution

- Creating and designing the website.

- Doing all of the rough work such as rough sketches and code snippets.

- Adding all of the assets such as sprites and images.

- Writing the game code, which includes the creation of all of the screens, logic and animations.

- Code testing and debugging.

Possible Challenges

For the string functions, I plan on checking the user's name to see if it is an appropriate length and edit it so the first letter is capitalized.

For the arrays, I plan on creating multiple to store the level clear times and information about the obstacles and platforms such as the speed and position.

For the sorting, I will incorporate it by sorting how fast the player has cleared the level from the shortest time to longest. This sorted array will then be displayed using jQuery as a board.

Screens

For this game, I plan on making 4 screens:

1. Start Screen - This screen will show the main title of the game and a small heading to tell the player to click a key to enter the game.

2. Main Game Screen This screen will have the actual game such as the sprites and obstacles. On this screen, the player will be able to move the character and send mail. This screen will also be recycled for the later levels.

3. Transition Screen This screen will transition from one level to another. It will show the player what level they are on and will increase the difficulty.

4. End Screen If the player loses all of their lives or runs out of time, they will be directed to this screen. This screen will show their score and a small game over message.

Concerns

I think the biggest challenge for this project is creating the logic for the game such as the moving platforms and obstacles. This also includes the movement for the player and the sending the mail to the homes. Additionally since I do not have much experience with Phaser, trying to figure out the various methods that Phaser has and implementing them will take some time as well.
